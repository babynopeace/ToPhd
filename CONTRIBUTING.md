# Contributing to ToPhd

感谢你帮助改进中国大陆申博信息工作流。

## 可以贡献什么

- 修正高校、科研院所和申请类型的通用规则；
- 改进导师筛选、证据核验和风险提示方式；
- 增加完全虚构或不可逆匿名化的使用示例；
- 改进空白模板、工作簿和生成脚本；
- 报告事实错误、流程缺口或隐私风险。

## 提交原则

1. 先判断内容是通用规则，还是只属于某位申请者的个案；
2. 个案信息不得写入 Skill；
3. 核心行为放在 `SKILL.md`，详细规则放在 `references/`；
4. 稳定、重复的操作才增加脚本；
5. 更新同名规则、脚本或模板时原位修改，不保留 `V1`、`V2`、`old` 或 `backup`；
6. 不夸大尚未实现的功能。

## 隐私要求

提交 Issue 或 Pull Request 前，请确认没有包含真实申请者的：

- 姓名、联系方式和个人路径；
- 简历、成绩、证书、照片和推荐信；
- 未公开论文、导师回复或项目资料；
- Office/PDF 作者、批注、隐藏文本和绝对路径元数据。

违反隐私要求的贡献可能被直接关闭或删除。

## 修改 Skill

- 保持 `SKILL.md` 简洁，使用命令式表达；
- 新增 reference 时必须从 `SKILL.md` 直接路由；
- 不在 `to-phd/` 内增加 README、变更日志或安装说明；
- 项目介绍和协作文档放在仓库根目录或 `docs/`；
- 修改后使用至少一个虚构场景重新测试。

## Pull Request 检查清单

- [ ] 修改属于可复用的国内申博规则
- [ ] 没有真实个人信息或本机路径
- [ ] 没有削弱隐私防火墙
- [ ] 没有把推断写成招生事实
- [ ] 文档和示例与当前实现一致
- [ ] 修改过的脚本已实际测试
