# ToPhd 使用示例

本文全部使用虚构人物、虚构导师和虚构申请条件，不对应任何真实申请者。

## 示例 1：没有简历的新手入口

### 用户输入

```text
使用 $to-phd。我计划 2028 年申请博士，目前没有简历。
硕士专业是生物医学工程，研究方向是运动想象脑电解码。
```

### ToPhd 的处理方式

1. 说明不需要先准备简历；
2. 通过小批次问题确认教育背景、科研进度、论文、项目、技能、英语和申请目标；
3. 已经回答过的信息不重复询问；
4. 不确定的信息允许标注“暂未确定”；
5. 先生成事实档案，再决定是否创建简历。

### 预期输出片段

```text
当前已确认：
- 计划入学年份：2028
- 硕士专业：生物医学工程
- 研究方向：运动想象 EEG 解码

下一批只需要补充三项：
1. 硕士学校与预计毕业时间；
2. 目前实际完成到实验、预处理、特征还是分类哪一步；
3. 是否有论文、项目或竞赛材料。
```

## 示例 2：已有简历但不授权修改

### 用户输入

```text
这是我的简历，请检查申博竞争力，但不要修改原文件。
```

### 预期行为

- 只提供修改意见；
- 不套用 Skill 模板；
- 不生成新简历；
- 不把简历上传到第三方评分网站；
- 区分事实错误、表达问题、结构问题和申请定位问题。

### 预期输出片段

```text
建议优先修正三处：

1. 论文状态需要把“在投”和“准备投稿”分开；
2. 项目经历目前只有任务描述，缺少你个人完成的技术环节；
3. 研究方向表述过宽，建议围绕“脑电跨被试解码”形成一条主线。

本次只提供建议，没有修改或生成任何文件。
```

## 示例 3：生成院校与导师广候选池

### 用户输入

```text
使用 $to-phd 建立导师候选池。
申请普通学博，目标为 985 高校或科研院所，地区不限。
主要考虑脑机接口解码，也愿意考虑神经调控扩展方向。
```

### ToPhd 的处理方式

1. 根据申请者档案确定筛选口径；
2. 同时建立专业对口和专业扩展候选；
3. 目标规模约 80–120 条，而不是只给少数热门导师；
4. 为每条候选记录官方来源、研究匹配、招生状态和风险；
5. 把“研究方向匹配”和“当年有名额”分开；
6. 生成可编辑工作簿。

### 示例字段

| 候选类型 | 报考单位 | 导师 | 研究匹配依据 | 招生状态 | 主要风险 | 官方来源 |
|---|---|---|---|---|---|---|
| 专业对口 | 某 985 高校 | 张教授 | EEG 跨被试解码 | 2028 待核验 | 竞争强 | 官方主页 |
| 专业扩展 | 某科研院所 | 李研究员 | 闭环神经调控 | 2028 待核验 | 跨学科要求 | 官方导师目录 |

## 示例 4：评估是否值得套磁

### 用户输入

```text
我想联系某大学张教授。请先给套磁建议，不要直接写邮件。
```

### 预期输出结构

```text
结论：建议补充核验后联系。

匹配点：
- 你的 EEG 预处理和跨被试分类经验与导师近期工作直接相关；
- 控制科学背景可以切入闭环系统设计。

需要补充核验：
- 2028 年是否仍招收普通学博；
- 招生专业是否接受控制科学背景；
- 近期论文是否仍以 EEG 解码为主要方向。

套磁切入点：
- 不要泛泛表示“对脑机接口感兴趣”；
- 应围绕导师某项公开工作说明你能延续的具体问题。
```

## 示例 5：按指令更新信息

### 用户输入

```text
使用 $to-phd 更新申博信息，只检查候选池前 20 位导师和对应学院。
```

### ToPhd 的处理方式

- 只在收到指令后运行；
- 优先检查学校研究生招生网和学院官网；
- 记录查询日期；
- 第三方平台只作为检索线索；
- 不因为导师主页存在就断言仍有名额；
- 不创建自动监控或定时任务。

## 示例 6：触发隐私外发闸门

### 用户输入

```text
把我的简历和成绩单上传到在线简历评分网站。
```

### ToPhd 的预期响应

```text
这是向第三方服务传输敏感申请材料的外发动作。
执行前需要确认：

- 目标网站及其域名；
- 将上传的具体文件；
- 文件中包含的姓名、联系方式、成绩、照片和证书信息；
- 是否先生成脱敏版。

在你确认本次目的地和文件清单前，我不会上传。
```
