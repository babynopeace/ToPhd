# 申请者档案工作流

## 目标

建立一份可持续更新的申请者事实底稿，供筛选、简历、联系导师和申请材料共同使用。不得把建议写成申请者已经具备的事实。

## 先选择入口

首次接触申请者时，先用一句话确认其属于哪种情况：

1. `有较完整简历`：已有能够反映教育、研究和成果的版本。
2. `没有简历`：尚未整理过经历，或不知道学术简历应该包含什么。
3. `只有旧简历或零散材料`：可能有求职简历、成绩单、成果列表、个人笔记或若干文件，但没有可直接用于申博的版本。

不得让“没有简历”成为阻塞条件，也不要要求小白先独自填完长表格。

## 入口 A：已有较完整简历

1. 先读取简历以及用户主动提供的成绩单、成果清单和既有档案。
2. 提取事实并填入 `../assets/applicant-profile-template.md`，不要默认简历中的所有说法已经核验。
3. 标出时间、成果状态、作者顺序、贡献和数字方面的矛盾或缺口。
4. 先向用户展示提取摘要，再集中询问最影响当前任务的 1 至 3 个问题。
5. 不重复询问材料中已经清楚回答的内容。
6. 完成建档后默认只给简历审阅与修改意见，不套用模板、不重排版、不生成新简历。只有用户明确要求编辑原简历时才修改现有文件；只有用户明确要求生成、重新制作或套用模板时才创建模板版简历。

## 入口 B：没有简历

先明确告诉申请者：没有简历不影响开始，可以通过对话建立事实底稿，之后再据此生成第一版学术简历。

采用四轮渐进访谈，每轮通常只问 1 至 3 个通俗问题；收到回答后先总结确认，再进入下一轮：

1. **目标和时间**：准备申请哪一届、预计何时硕士毕业、目前大概想研究什么。不要求立即给出精确研究方向。
2. **教育背景与英语**：本科和硕士学校、专业、时间、成绩或排名是否有可提供的证明；是否参加过CET-4、CET-6、雅思、托福等英语考试。
3. **经历盘点**：硕士课题、实验室工作、课程项目、竞赛、实习、助研或其他真正做过的事情。用“你具体负责了什么、用了什么方法、得到什么结果”帮助回忆。
4. **成果与约束**：论文或其他产出、技能、奖项、目标地区、时间和明确排除项。

第2轮不得只问成绩单而漏问英语。通常把第三个问题组织为：“目前成绩或排名怎样，是否有成绩单？参加过哪些英语考试，成绩是多少；成绩还没公布也可以说明预计公布时间。”

允许回答“不知道”“没有”“还没想好”。解释必要术语，并在问题中给出简短例子，但不要诱导用户声称不存在的经历。

如果申请者没有论文、奖项或正式科研经历，如实记录。可以识别毕业论文、课程项目或实习中可迁移的研究能力，但不能把它们改称论文或正式科研成果。

## 入口 C：旧简历或零散材料

1. 先把所有材料视为线索，提取能够相互印证的事实。
2. 把求职简历中的经历重新拆为研究问题、本人职责、方法、结果和证据，不直接沿用商业化或夸张表述。
3. 列出已覆盖信息和缺失信息，只访谈缺失且影响当前任务的部分。
4. 完成档案后默认先提供修改意见；只有用户明确要求编辑原简历时才修改现有文件，明确要求生成新简历时才套用模板。

## 处理成绩单、证书和英语材料

1. 用户表示有材料但不知道放在哪里时，明确说明可直接附加到当前任务，或保存到案例工作区的 `materials/` 分类目录，不让上传位置成为阻塞项。
2. 从成绩单和考试证书中提取申请需要的字段，不在回复或档案中重复学号、准考证号、验证码等无关敏感信息。
3. 对英语考试分别记录考试类型、考试时间、总分、分项成绩（如有必要）、证据和状态。
4. 把尚未出分的考试记为“已参加、成绩待公布”，并记录预计公布时间；不得写成已通过或预估分数。
5. 对扫描件、手写内容或低置信度识别结果标记“待用户确认”。用户更正后使用更正值，并废止错误值。

## 建档通用规则

1. 使用 `../assets/applicant-profile-template.md` 建立或补全结构。
2. 为关键信息标记状态：`已确认`、`待确认`、`缺失`、`不适用`。
3. 记录信息来自哪份材料或哪次用户确认；存在矛盾时保留两种说法并请求确认。
4. 只询问当前任务的阻塞项，其余缺口放入“待补充信息”。
5. 不要因为档案未填满而阻止后续工作；明确说明哪些结论会因此降低可信度。

## 最小可用档案

开始初步规划通常只需确认：申请周期、预计毕业时间、教育背景、当前研究或经历的大致范围、英语考试状态、地域或其他硬性约束。其余信息可以在后续任务中逐步补全。

进行精确资格筛选、生成正式简历或撰写联系邮件前，再补齐这些任务必需的证据。

## 必要字段

- 申请周期、预计毕业时间、申请类型。
- 本科与硕士教育背景、专业、成绩或排名及其可证明状态。
- 硕士培养方式、学位状态及毕业要求。
- 研究兴趣：研究问题、对象、方法和希望继续发展的方向。
- 每段研究经历的背景、本人职责、方法、产出、证据和合作者。
- 论文、专利、软件、数据集、会议报告等成果及准确状态。
- 技能与课程；CET-4、CET-6、雅思、托福等语言考试的时间、分数、待出分状态及证明。
- 奖项、项目和学术服务。
- 目标地域、学校层级、院系类型、导师偏好和硬性排除项。
- 时间、经费、家庭、健康等会影响选择的约束；只记录用户愿意提供且确有必要的内容。
- 推荐人准备情况、现有材料和当前申请进度。

## 提取研究经历

把“课题整体做了什么”和“申请者本人做了什么”分开。优先提取：

- 申请者提出或负责的问题。
- 使用或开发的方法。
- 数据、实验、分析或工程工作的规模。
- 结果、失败经验和可验证产出。
- 与目标博士方向之间的连接。

缺少量化信息时，提出可回答的问题，不编造数字。

## 输出档案摘要

完成档案后，给出：

1. 一段不超过 150 字的申请者定位。
2. 3 至 5 项已证实优势。
3. 影响申请的主要风险或缺口。
4. 当前最适合优先推进的三项行动。
5. 尚未确认但可能改变策略的问题。

对于没有简历的小白，额外说明：目前已经完成到哪一轮、下一轮只需要回答什么、何时具备生成第一版简历的条件。

## 隐私规则

- 默认不记录身份证号、银行卡信息、系统密码、完整家庭住址等高敏感数据。
- 联系方式只在具体材料确有需要时使用，不复制到研究或筛选记录。
- 分享案例用于测试前，优先制作去标识版本。
